# SCIF Driver API — System CPU Reference

The SCIF (Sensor Controller Interface) driver is a thin layer between the System CPU application and the Sensor Controller hardware. It is split into two parts:

- **Generic framework** (`scif_framework.c/.h`): handles AUX RAM loading, interrupt routing, task control state machine. Copied verbatim from the SCS installation.
- **Project-specific driver** (`scif.c/.h`): contains the compiled AUX RAM image, task data struct definitions, and any resource-specific helpers (e.g., RTC functions). Generated by Sensor Controller Studio.

## Initialization and Teardown

```c
// Load AUX RAM image, configure pins, register interrupt vectors
// Call once at startup before starting any tasks
SCIF_RESULT_T scifInit(const SCIF_DATA_T* pScifDriverSetup);

// Unload firmware, release pins, disable interrupts
// Call when done with the Sensor Controller
SCIF_RESULT_T scifUninit(void);
```

Return values: `SCIF_SUCCESS` (0) or `SCIF_ILLEGAL_OPERATION`.

## Task Control

```c
// Start one or more tasks (non-blocking)
// bvTaskIds: bit-vector of task IDs, e.g. BV(SCIF_ADC_INTERVAL_POLLER_TASK_ID)
// Returns immediately; ctrlReadyCallback fires when SC has executed Initialize
SCIF_RESULT_T scifStartTasksNbl(uint32_t bvTaskIds);

// Stop one or more tasks (non-blocking)
// Returns immediately; ctrlReadyCallback fires when SC has executed Terminate
SCIF_RESULT_T scifStopTasksNbl(uint32_t bvTaskIds);

// Wait synchronously for a pending task control operation to complete
// Blocks until the READY interrupt fires
SCIF_RESULT_T scifWaitOnNbl(uint32_t timeoutMs);

// Start tasks synchronously (start + wait)
SCIF_RESULT_T scifStartTasksSync(uint32_t bvTaskIds, uint32_t timeoutMs);

// Stop tasks synchronously (stop + wait)
SCIF_RESULT_T scifStopTasksSync(uint32_t bvTaskIds, uint32_t timeoutMs);

// Reset all tasks and reinitialize the SC firmware
SCIF_RESULT_T scifResetTaskStructs(uint32_t bvTaskIds,
                                   uint32_t bvTaskStructTypes);
```

`BV(n)` = `(1u << (n))`, defined in `scif_framework.h` or locally.

## Interrupt / Alert Handling

```c
// Register a callback invoked when ctrlReady (SWEV0) fires
// Must be registered before scifInit()
void scifOsalRegisterCtrlReadyCallback(void (*callback)(void));

// Register a callback invoked when taskAlert (SWEV1) fires
// Must be registered before scifInit()
void scifOsalRegisterTaskAlertCallback(void (*callback)(void));

// Inside the alert callback: clear the ALERT interrupt source in AUX_EVCTL
void scifClearAlertIntSource(void);

// Inside the alert callback: acknowledge and resume the Sensor Controller
// Must be called last, after reading all output data
void scifAckAlertEvents(void);

// Get bitmask of tasks that generated the current alert
// Useful when multiple tasks share one alert callback
uint32_t scifGetAlertEvents(void);
```

**Alert callback pattern:**
```c
void taskAlertCallback(void) {
    scifClearAlertIntSource();
    // Read output data here — SC is paused, AUX RAM is stable
    g_adcValue    = scifTaskData.adcIntervalPoller.output.adcValue;
    g_sampleCount = scifTaskData.adcIntervalPoller.output.sampleCount;
    g_newSample   = true;
    scifAckAlertEvents();   // must be last
}
```

### Interrupt Enable/Disable

```c
// Enable the ALERT interrupt (INT_AON_AUX_SWEV1)
void scifOsalEnableTaskAlertInt(void);

// Disable the ALERT interrupt
void scifOsalDisableTaskAlertInt(void);
```

## AUX Domain Power Management

```c
// Force AUX power domain on (required for OSAL "None" to access AUX RAM)
// ~0.5 µA extra quiescent current
void scifOsalEnableAuxDomainAccess(void);

// Allow AUX power domain to turn off
// Must call scifOsalEnableAuxDomainAccess() again before accessing AUX RAM
void scifOsalDisableAuxDomainAccess(void);
```

For low-power applications, bracket every AUX RAM access with Enable/Disable. For simplicity (this example), call Enable once at startup.

## Task Data Access

The generated `scif.h` defines `scifTaskData` as a volatile pointer to AUX RAM:

```c
#define SCIF_TASK_DATA_ADDR  0x400E01E0UL   // PLACEHOLDER – use SCS-generated value
#define scifTaskData  (*((volatile SCIF_TASK_DATA_T*) SCIF_TASK_DATA_ADDR))
```

Accessing `scifTaskData` is safe inside the ALERT callback (SC is paused). In the main loop, guard reads with interrupt disable or use the `g_newSample` flag pattern.

**Examples:**
```c
// In alert callback:
uint16_t raw = scifTaskData.adcIntervalPoller.output.adcValue;

// Write input/cfg before starting task:
scifTaskData.myTask.cfg.thresholdLow  = 500;
scifTaskData.myTask.cfg.thresholdHigh = 3000;
```

## SCIF_DATA_T — Driver Setup Structure

The generated `scif.c` defines this constant, passed to `scifInit()`:

```c
const SCIF_DATA_T scifDriverSetup = {
    (volatile SCIF_INT_DATA_T*)  pIntData,              // AUX RAM: framework internal data
    (volatile SCIF_TASK_CTRL_T*) pTaskCtrl,             // AUX RAM: task control block
    (volatile uint16_t*)         pTaskExecuteSchedule,  // AUX RAM: execute schedule table
    0x0000,                                             // bvDirtyTasks (always 0 on init)
    sizeof(pAuxRamImage),                               // image size in bytes
    pAuxRamImage,                                       // compiled SC bytecode
    pScifTaskDataStructInfoLut,                         // task struct size/address table
    pAuxIoIndexToMcuIocfgOffsetLut,                    // AUXIO → MCU IOCFG offset table
    scifTaskResourceInit,                               // called by scifInit()
    scifTaskResourceUninit,                             // called by scifUninit()
    NULL, NULL, NULL                                    // RTL logging (not used)
};
```

All AUX RAM addresses are **PLACEHOLDER** values until you generate code with SCS.

## RTC Scheduling Functions (generated into scif.c)

These are placed in the project-specific `scif.c` by the "RTC-Based Execution Scheduling" resource:

```c
// Set specific start time and period
// tickStart/tickPeriod: 16.16 fixed-point seconds
void scifStartRtcTicks(uint32_t tickStart, uint32_t tickPeriod);

// Start scheduling ~128 µs from now (most common use)
void scifStartRtcTicksNow(uint32_t tickPeriod);

// Disable AON_RTC channel 2 scheduling
void scifStopRtcTicks(void);
```

Call `scifStartRtcTicksNow(ADC_TICK_PERIOD)` **before** `scifStartTasksNbl()`.

## Complete Startup Sequence

```c
// 1. Power up PERIPH domain (for GPIO clocks)
PRCMPowerDomainOn(PRCM_DOMAIN_PERIPH);
while (PRCMPowerDomainStatus(PRCM_DOMAIN_PERIPH) != PRCM_DOMAIN_POWER_ON);
PRCMPeripheralRunEnable(PRCM_PERIPH_GPIO);
PRCMLoadSet();
while (!PRCMLoadGet());

// 2. Keep AUX domain on (OSAL None requirement)
scifOsalEnableAuxDomainAccess();

// 3. Register callbacks (before scifInit)
scifOsalRegisterCtrlReadyCallback(ctrlReadyCallback);
scifOsalRegisterTaskAlertCallback(taskAlertCallback);

// 4. Initialize SCIF driver (loads AUX RAM image)
scifInit(&scifDriverSetup);

// 5. Write any cfg/input data the SC needs before start
// scifTaskData.myTask.cfg.someParam = 42;

// 6. Start RTC ticks (must come before starting tasks if using RTC scheduling)
AONRTCEnable();
scifStartRtcTicksNow(0x000A0000);   // 10 Hz

// 7. Enable global interrupts
IntMasterEnable();

// 8. Start tasks
scifStartTasksNbl(BV(SCIF_ADC_INTERVAL_POLLER_TASK_ID));
```

---

## Complete Shutdown Sequence

```c
// 1. Stop tasks (non-blocking)
scifStopTasksNbl(BV(SCIF_ADC_INTERVAL_POLLER_TASK_ID));
scifWaitOnNbl(1000);   // wait up to 1000 ms for Terminate to complete

// 2. Stop RTC scheduling
scifStopRtcTicks();

// 3. Uninitialize SCIF driver
scifUninit();

// 4. Allow AUX to power down (optional)
scifOsalDisableAuxDomainAccess();
```
