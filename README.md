## Quick Start

1. Install **Sensor Controller Studio** and a **SimpleLink SDK** (see `05_project_setup.md`)
2. Copy the framework files from SCS to your project (`scif_framework.c/h` plus the OSAL pair matching your OS choice: `scif_osal_none.c/h`, `scif_osal_tirtos.c/h`, or `scif_osal_tidpl.c/h`)
3. Write your task in a `.scp` file, click **Generate Code** → copies `scif.c` and `scif.h`
4. Write `main.c` using the SCIF API (see `03_scif_driver_api.md`)
5. Build in CCS and flash

## This Project (`adc_data_logger_launchpad`)

An ADC data logger for the LaunchPad: the Sensor Controller samples DIO23 at 10 Hz and stores values in a 128-entry circular buffer in AUX RAM. The System CPU does **not** receive alerts — it wakes every 10 s, compares `output.head` against a local `tail` index, and drains new samples over UART (57600 8-N-1).

```
examples/
├── adc_data_logger_launchpad.scp   ← Open in Sensor Controller Studio; click Generate Code
├── source/
│   ├── main.c                      ← Bare-metal (OSAL None) System CPU application
│   ├── main_tirtos.c               ← TI-RTOS variant
│   ├── ex_include.h / ex_include_tirtos.h
│   ├── scif.c/.h                   ← Generated by SCS — regenerate after .scp changes
│   ├── scif_framework.c/.h
│   └── scif_osal_tirtos.c/.h       ← OSAL variant matching the .scp's OS setting
└── projects/
    ├── ccs/         ccs_tirtos/    ← CCS project specs (bare-metal and TI-RTOS)
    └── iar/         iar_tirtos/    ← IAR project files
```

The shipped `scif.c`/`scif.h` are generated for **TI-RTOS** on **CC2642R1F**. To build `main.c` (bare-metal), open the `.scp` in SCS, switch *Operating System* to *None*, click **Generate Code**, and replace `scif.c`/`scif.h` plus the OSAL pair.
